# 01-university
Repository dedicated to organizing all university content, including classes, notes, and exercises. Structured by semester, subject, and activities, it provides a clear learning path, tracks academic progress, and centralizes study materials for easy access and continuous development.


# 📘 Repository Naming Pattern

Repositories follow a structured naming pattern to ensure organization, clarity, and easy identification of content.

---

## 🧩 Format

```bash
01-SM1-poo-class
```

---

## 📌 Meaning

* **01** → repository number
* **SM1** → semester (1st semester)
* **poo** → subject (Object-Oriented Programming)
* **class** → activity name

---

## 📂 Content Structure

Each repository represents a specific activity within a subject and semester.

### 📁 Example:

```bash
src/
 ├── index.js
 ├── class.js

README.md
```

---

## 📌 Naming Rules

* Use lowercase letters only
* Separate words with hyphens (`-`)
* Keep names clear and descriptive
* Follow sequential numbering

---

## 🎯 Objective

* Organize activities by semester and subject
* Make content easy to find and understand
* Maintain a structured learning history

---

## 🚀 Note

This pattern will be maintained for future semesters:

```bash
02-SM2-data-structures-list
03-SM3-database-project
```

Ensuring consistency across all repositories.

---

# 📘 Padrão de Nomenclatura dos Repositórios

Os repositórios seguem um padrão estruturado para garantir organização, clareza e fácil identificação dos conteúdos.

---

## 🧩 Formato

```bash
01-SM1-poo-class
```

---

## 📌 Significado

* **01** → número do repositório
* **SM1** → semestre (1º semestre)
* **poo** → matéria (Programação Orientada a Objetos)
* **class** → nome da atividade

---

## 📂 Estrutura do Conteúdo

Cada repositório representa uma atividade específica dentro de uma matéria e semestre.

### 📁 Exemplo:

```bash
src/
 ├── index.js
 ├── class.js

README.md
```

---

## 📌 Regras de Nomeação

* Utilizar apenas letras minúsculas
* Separar palavras com hífen (`-`)
* Nomear de forma clara e objetiva
* Seguir a sequência numérica

---

## 🎯 Objetivo

* Organizar atividades por semestre e matéria
* Facilitar localização e entendimento dos conteúdos
* Manter um histórico estruturado dos estudos

---

## 🚀 Observação

O padrão será mantido para os próximos semestres:

```bash
02-SM2-estrutura-dados-lista
03-SM3-banco-dados-projeto
```

Garantindo consistência em toda a organização.
